# Phase 1.1: Critical Fixes & Unification Completion

Implementation of system health checks and async data offloading.

## Context
Project unification is largely complete, but performance and observability gaps remain. Synchronous Pandas calls are blocking the main event loop, and there is no easy way for users to verify their installation.

## Tasks

<task id="doctor-impl">
**amberclaw doctor**: Implementation of system health check.
- Create `amberclaw/cli/doctor.py` with health checks (API keys, config, workspace).
- Integrate `doctor` into `amberclaw/cli/commands.py`.
</task>

<task id="pandas-offload">
**Pandas offloading**: Wrap data operations in `asyncio.to_thread`.
- Audit `amberclaw/agent/tools/data_*.py` for synchronous Pandas calls.
- Refactor tools to use `asyncio.to_thread` for `pd.read_csv` and agent execution.
</task>

<task id="final-cleanup">
**Final cleanup**: Verify no dead code in `amberclaw/data`.
- Remove any unused legacy files found during the audit.
</task>
