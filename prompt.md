### 🚀 The Command
**"Reconstruct and unify [FEATURE_NAME] into the AmberClaw core using the Production-Grade Feature Protocol."**

### 🧠 Instructions
1. **Research first**: Dig into every source file and legacy module to extract the core intellectual property behind [FEATURE_NAME]. Never assume anything — verify architectural fit before writing code.
2. **Unified reconstruction**:
    - Place feature logic under `amberclaw/features/` and performance-critical code under `amberclaw/engine/`.
    - Stick to **pure Python** so the result runs everywhere — Android, Linux, and Windows.
    - After migrating code, immediately delete any unused or duplicate legacy directories.

3. **Antigravity execution**:
    - Invoke the appropriate **Antigravity skills** (such as `@python-pro`, `@senior-fullstack`, `@code-reviewer`) before every action.
    - Apply **@caveman** token optimization throughout internal reasoning.
4. **Production excellence**:
    - Keep the unified CLI-first interface (Rich-based terminal UI).
    - All code must be production-ready, fully type-checked, and self-documenting.
    - Update `README.md` to reflect any new capabilities.

### 🎯 Acceptance Criteria
- Zero leftover legacy `feature/` folders.
